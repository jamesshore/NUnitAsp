NUnitAsp allows you to test ASP.NET web pages. It's freely available under the MIT license.

NUnitAsp acts as a client to a regular web server running ASP.NET. Using NUnitAsp, you may download web pages and work with the controls on it as if they were actual ASP.NET controls.

public void TestExample()
{
   LabelTester label = new LabelTester("textLabel", CurrentWebForm);
   LinkButtonTester link = new LinkButtonTester("linkButton", CurrentWebForm);

   AssertEquals("first page", "FirstExample", CurrentWebForm.AspId);
   AssertEquals("label", "Example Label", label.Text);
   link.Click();
   AssertEquals("second page", "SecondExample", CurrentWebForm.AspId);
}

NUnitAsp can test complex web sites involving multiple pages and nested controls. The common ASP.NET controls (see complete list below) are supported, and support for additional controls is easy to add.

NUnitAsp is a tool for testing server-side ASP.NET logic only. It does not support JavaScript or other client-specific behavior, although support for this may be added in the future.

NUnitAsp is an extension for NUnit, which is part of the xUnit family of freely-available test-first programming products.

Latest Release: Version 1.1

NUnitAsp 1.1 is capable of testing most ASP.NET controls. It is being used for real-world applications, but its API is subject to change. We recommend downloading and trying NUnitAsp, and contributing new controls, but if you use it, expect to modify your tests to keep up with API changes.

This release does not include complete documentation. Use the mailing list if you need help.
