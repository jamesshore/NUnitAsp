Installation Guide

This guide describes how to install NUnitAsp. It assumes that you are using Windows 2000 or Windows XP and Microsoft Internet Information Server as your web server. NUnitAsp will work with any server that supports ASP.NET, but you'll have to modify these instructions to work in your environment. If you do, please let us know what you did! We'll expand this guide to include your information.

Installation

Installing NUnitAsp requires only a few steps:

  1. Download and unzip NUnitAsp.
    1. You've probably already done this step, but if not, read on.
    2. NUnitAsp may be downloaded from the NUnitAsp home page, nunitasp.sourceforge.net.
    3. Unzip the NUnitAsp package into a directory of your choice, such as "C:\Program Files\NUnitAsp"

  2. Install NUnit 2.0.
    1. We include an install file for NUnit 2.0 in the NUnitAsp distribution. It's in the bin directory. (For example, if you unzipped NUnitAsp into C:\Program Files\NUnitAsp, it's in C:\Program Files\NUnitAsp\bin.)

  3. Create an NUnitAsp virtual directory in IIS.
    1. Right-click "My Computer" and select "Manage"
    2. Expand "Services and Applications" -> "Internet Information Services" -> "Default Web Site"
    3. Right-click "Default Web Site" and select "New" -> "Virtual Directory"
    4. Enter the following information into the Virtual Directory Creation Wizard:
      • Alias: NUnitAsp
      • Directory: installation directory (For example, if you unzipped NUnitAsp into "C:\Program Files\NUnitAsp", you would enter "C:\Program Files\NUnitAsp" here.)
      • Access Permissions: "Read" should be checked. The other options should not be checked.

  4. Test Installation.
    1. Verify that you can visit http://localhost/NUnitAsp/web/dtd/xhtml1-transitional.dtd.
      Note: If you have XMLSpy installed, you may get a false error during this step. See below.
      Note 2: Some browsers, including Netscape and Phoenix, display a download window rather than showing you the DTD. This is also a sign of success.

Error While Testing DTD

If you have XMLSpy installed, you may see the following error when you test the installation:

The XML page cannot be displayed

Cannot view XML input using style sheet. Please correct the error and then click the Refresh button, or try again later. Cannot have a DTD declaration outside of a DTD. Error processing resource 'http://localhost/NUnitAsp/web/dtd/xhtml1-transitional.dtd'. Line 51, Position 12

<!ENTITY % HTMLlat1 PUBLIC
-----------^

You may ignore this error. Even though your browser can't read the file, the web server is serving it correctly.

This error reportedly occurs because XMLSpy causes Internet Explorer to see DTD files as XML files. Thanks to Peter Wijntjes for the report.
