Frequently Asked Questions

The NUnitAsp team sees frequently asked questions as indicators of problems with NUnitAsp and its documentation. We strive to eliminate FAQs, either by improving the documentation or (preferably) improving NUnitAsp itself. As a result, there's not much in the way of FAQs.

This also isn't the best place to start for general information about NUnitAsp. For an overview, see the NUnitAsp QuickStart Guide. For a detailed tutorial, see the NUnitAsp Tutorial. For installation issues, see the Installation Guide. NUnitAsp is available for download at the NUnitAsp web site.

Q: When will NUnitAsp stop requiring XHTML?

A: XHTML is still required, and may always be, but with NUnitAsp v1.3 we now automatically compensate for common XHTML problems. We no longer require a specific DOCTYPE tag or closed tags. Visual Studio .NET HTML will work without modification.

Q: When I try to compile NUnitAsp, I get a "cryptographic failure" error. What's wrong?

A: The NUnitAsp DLL is strong-named, which means it must be signed by a cryptographic key. If you try to compile without a key, you'll get this error message:

Cryptographic failure while signing assembly 'C:\projects\NUnitAsp\source\NUnitAsp\obj\Debug\NUnitAsp.dll' -- 'The key container name 'NUnitAsp' does not exist'

We don't include a key in the distribution, but the NUnitAsp build script automatically generates the correct kind of key for you. Open a Visual Studio .NET command prompt and type 'nant' in the root NUnitAsp directory to run it. You only need to do this the first time you compile.

Q: I want to send you a patch. What should I do?

A: Great! We appreciate your enthusiasm and support. The best way you can help is to send us a test case that demonstrates your bug or feature. In your patch, integrate your test case into the existing tests. If you can, download the latest NUnitAsp code from CVS first (see the NUnitAsp Wiki for more information). The easier it is for us to run your test, the sooner we'll get to your suggestion. Once your patch is ready, upload it to the SourceForge patch manager, and then post an explanation to the developer mailing list.

(End of FAQ. Last updated for v1.3.)
