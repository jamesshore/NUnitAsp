To release NUnitAsp, follow these steps. You'll need admin priviliges.

  • Update the web site content
    • Check out the latest files from CVS
    • Update the "current version" info at the bottom of website/index.html
    • Update the current version in website/download.html. Don't forget to update the HTML link.
    • Update changes.txt. Don't forget to update the "changes in CVS" to "changes in v#.#"

  • Create the release
    • Announce release freeze on nunitasp-devl
    • Update source\NUnitAsp\AssemblyInfo?.cs with the correct version info (e.g., "[assembly: AssemblyVersion?("1.1.1.*")]"
    • Use NAnt to create the release: nant clean package
    • Rename the release file so that includes the correct version info (e.g., "NUnitAsp-1.1.1.zip")
    • Double-check the release file and web site
    • Commit your changes to CVS
    • Tag the CVS tree with the name of the release (e.g., "Release1-1-1")

  • Upload the release
    • Upload the release file to ftp://upload.sourceforge.net/incoming (anonymous login)
    • Register release with SourceForge? at https://sourceforge.net/project/admin/newrelease.php?package_id=43375&group_id=49940
    • Release name: "NUnitAsp-x.x" (e.g., NUnitAsp-1.1.1)
    • Processor: i386
    • File Type: .zip
    • After finishing, double-check that the link on website/download.html works.

  • Update the web site
    • Log into nunitasp.sf.net shell server using SSH: ssh -l username nunitasp.sf.net
    • Change to the NUnitAsp project's directory: /home/groups/n/nu/nunitasp
    • Rename ('mv') htdocs to htdocs.v#.# (e.g., htdocs.v1.4)
    • Delete ('rm -rf') the old htdocs backup
    • At Windows command prompt, copy the docs to the server: scp -r doc username@nunitasp.sf.net:/home/groups/n/nu/nunitasp/htdocs
    • At Unix (ssh) command prompt, update doc permissions: chmod -R 775 htdocs
    • Check that the update worked: http://nunitasp.sf.net

  • Announce the release
    • Create a news item on SourceForge.
    • Send email to these addresses (minimum): nunitasp-announce@lists.sourceforge.net, nunitasp-devl@lists.sourceforge.net, nunitasp-users@lists.sourceforge.net, agile-testing@yahoogroups.com, extremeprogramming@yahoogroups.com, testdrivendevelopment@yahoogroups.com, testfirstuserinterfaces@yahoogroups.com

Sample release email: (Subject: NUnitAsp 1.5.1 released)

NUnitAsp v1.5.1 is now available for download. NUnitAsp is a tool for test-driven development of ASP.NET web pages.

Version 1.5 includes a modest collection of improvements over version 1.4. The most important new feature is improved custom tester support. We've exposed a number of methods needed for writing custom testers and we've created a new HtmlTag class that's easy to use and understand. Even if you aren't writing custom testers, you'll appreciate 1.5's improved error reporting and API documentation. Particularly handy is the new ability to report server-side ASP.NET exceptions directly within test failure messages.

Version 1.5.1 is nearly identical to version 1.5. It requires NUnit 2.2 rather than NUnit 2.1.

NUnitAsp v1.5.1 is available for download at: http://prdownloads.sourceforge.net/nunitasp/NUnitAsp-1.5.1.zip?download.

A detailed change log may be found at: http://nunitasp.sourceforge.net/changes.txt

The NUnitAsp website is at http://nunitasp.sourceforge.net.

Jim Shore
Project Coordinator
