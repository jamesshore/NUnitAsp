Contribute

If you have modified NUnitAsp or developed your own testers, let us know! We will integrate your changes into NUnitAsp so others can use them and you don't have to maintain them. Be sure to follow these guidelines when sending in changes:

  • Start with the latest code in CVS.
  • Always write tests for your changes.
  • Match your coding style to the existing NUnitAsp style.
  • Submit your changes to the SourceForge Patches tracker or the nunitasp-devl mailing list.

Patches that don't follow these guidelines will take longer to be integrated and may even be rejected entirely.

We maintain a web site for contributors. See that web site for details on getting the latest code out of CVS, setting up your development environment, and ideas for future changes.

If you're not a coder, but you'd like to contribute, we can still use your help! Graphic design, writing, and web development skills are also needed. See the contributors' web site for details.
