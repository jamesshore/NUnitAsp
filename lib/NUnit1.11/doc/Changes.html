NUnit Changes

Changes in v1.11 (25 February 2002)

General

  • .Net RTM Version.
  • General code maintenance.
  • Obsoleted classes that use Java specific concept of reloading.
  • Added Reloading. GUI runner will release test assembly as soon as tests are completed. Tried as much as possible to introduce as few breaking changes and as few new requirements as possible. If you have built your own runners or have extended parts of the framework, you might want to look at the ReloadingChanges.txt. Many thanks to Bob Davison for his input and ideas on this topic, and to Ethan Smith for doing all the work on this release.

GUI

  • Cosmetic Fixes.
  • Fixed dynamic layout issues.
  • New Progress Bar :)
  • Changed scope of most GUI components to internal.

Changes in v1.10 (26 June 2001)

General

  • .Net beta 2 version.

GUI

Changes in v1.9 (21 Apr 2001)

Framework

  • Added TestSuite.Name property.

General

  • Incorporated all JUnit 3.6 changes and bug fixes except for GUI changes.

GUI

Documentation

  • Added documentation about the preferences supported by TestRunners.
  • Updated the FAQ

Changes in v1.8 (7 Mar 2001)

General

  • Turned into a Visual Studio.Net solution. Updated to include enhancements ported from JUnit 3.5.

Framework

  • Added TestSuite.addTestSuite(Class testClass)
    This method allows to create a TestSuite with a class containing test cases directly.
    Instead of writing suite.addTest(new TestSuite(AssertTest.class)) you can now write suite.addTestSuite(AssertTest.class).

TestRunner

  • The Text TestRunner now shows the summary at the very end so that you don't have to scroll back.

General

Documentation

  • Minor documentation corrections. XML links improved.

Samples

Documentation

Samples

Changes in v1.5 (15 Oct 2000)

  • Red/green progress bar (from Jim Newkirk)

Changes in v1.4.1 (7 Oct 2000)

  • Icons for test suite browser. Run button runs tests, double clicking expands tests. (from Jim Newkirk)

Changes in v1.4 (27 Sep 2000)

  • A menu with a test suite browser on it. Open the browser and double click anywhere in the tree of test suites to run everything under that point in the hierarchy. (from Jim Newkirk)
  • An About box.

Changes in v1.3 (23 Sep 2000)

A much improved GUI, courtesy of Ethan Smith and Jim Newkirk.
  • You can now browse to DLLs or EXEs. NUnit will identify all the classes that have TestSuites implemented and give a dropdown letting you choose which TestSuite to run.
  • Failures, Errors, and error and text outputs all have their own tabs.
  • Double-clicking on a failure shows a stack trace.

Changes in v1.2 (18 Sep 2000)

  • First cut at a GUI, courtesy of Jim Newkirk

Changes from v1.1 (9 Sep 2000)

  • .Net style casing on all method names (thanks to Ethan Smith) Assertion.cs changed to Assert.cs to match class name.

Changes from v1.0 (1 Sep 2000)

  • Converted to properties where sensible. Note that NUnit now looks for tests in a static property named Suite, not a method named suite().
  • Added some assembly info.
  • Added some readonly optimisations.
  • Fixed a bug where the shown identifier of failures and errors didn't increment in the output.
  • Added a VB test example. The power of .Net really showed here. No code changes were necessary to make the VB sample work.
  • Added the junit licence as required.
