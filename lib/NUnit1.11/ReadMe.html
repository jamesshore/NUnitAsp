NUnit has been ported from JUnit, by Kent Beck and Erich Gamma. Seewww.junit.org.

Installation

Use the SetupNUnit.msi Windows Installer file. Then run NUnit from the NUnit program group on the Start menu. This will launch the GUI. Also installed into the NUnit directory is NUnitConsole.exe, which is a text-only version. Below are the manual installation steps for installing NUnit from the compiled source:
  1. Copy NUnit*.exe and NUnitCore.dll to your desired location.
  2. Test the installation by using either the batch NUnitConsole or the graphical NUnitGUI tool to run the SampleAll.dll that comes with this release. All the tests should pass OK.
    • for the batch NUnitConsole.exe type:
          NUnitConsole NUnit.Samples.AllTests,SampleAll.dll
    • for the graphical NUnit type:
          NUnitGUI NUnit.Samples.AllTests,SampleAll.dll

To use NUnit in your own projects reference the NUnitCore project (Add Reference in VisualStudio.Net or /r from the csc or vbc compilers. If you'd like to launch the NUnitGUI or Console directly from within your project (which helps with debugging), also reference to one of NUnitGUI.exe or NUnitConsole.exe, and call the NUnit method NUnit.Top.Main("MyTestNameSpace.MyTestClass").

Refer to Installing NUnit in IIS for notes about installing NUnit as a WebService in IIS (experimental).

Getting Started

To get started with unit testing and NUnit read the article: TestInfected - Programmers Love Writing Tests.
This article demonstrates the development process with NUnit in the context of multiple currency arithmetic. The corresponding source code is in SampleMoney.

You will find additional samples in the NUnit.Samples namespace:

  • SimpleTest.cs - some simple test cases in C#
  • SimpleVBTest.vb - some simple test cases in VB
  • SimpleJSTest.js - some simple test cases in JScript
  • ArrayList.cs - test cases for System.Collections.ArrayList (in C#)

Which TestRunner to use?

NUnit comes with two different TestRunners:
  • NUnitConsole.exe textual TestRunner - it is the fastest to launch and can be used when you don't need a red green success indication.
  • NUnitGUI.exe graphical TestRunner - shows a simple graphical dialog to enter the test and provide some graphical progress indication.

Extending NUnit

Examples of possible NUnit extensions can be found in the NUnit.Extensionsnamespace:
  • TestDecorator - A decorator for Test. You can use it as the base class for implementing decorators to extend test cases.
  • ActiveTestSuite - A TestSuite which runs each test in a separate thread.
  • TestSetup - A Decorator to set up and tear down additional fixture state. Subclass TestSetup and insert it into your tests when you want to set up additional state once before the tests are run.
  • ExceptionTestCase - A TestCase that expects a particular Exception to be thrown.
